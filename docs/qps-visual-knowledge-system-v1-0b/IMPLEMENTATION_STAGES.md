# QPS Visual Knowledge System v1.0b — Implementation Stages

## Objective

Transform QPS master-slide ecosystems into:
- semantic engineering artefacts
- navigable HTML knowledge surfaces
- YAML-driven render systems
- PPTX/PDF/ODP-compatible visual systems
- ontology-aware engineering decks

while preserving:
- dense engineering cognition
- visual lineage
- review overlays
- institutional aesthetic identity
- SBS and naming SSOT logic

---

# STAGE 0 — Canonical Source Preservation

## Goal
Preserve user-authored slides and visual lineage before aggressive transformation.

## Inputs
- PPTX
- PDF
- slide-image ZIP
- ODP
- PPTM

## Outputs
- source-image galleries
- OCR extraction cache
- lineage metadata
- canonical deck registry

## KPI
- 100% source retention
- no silent slide deletion
- lineage traceable to original deck
