# CODEX + ABACUS Block Library — DMAIC Iteration 002

Status: proposed architecture baseline  
Scope: reusable code/document block library, YAML/JSON SSOT, Markdown readability, Word `.dotm` rendering, Excel review, and runtime validation path.

## 1. Decision

Adopt a three-layer content model:

```text
Layer 1 — Human Authoring: Markdown with fenced code blocks and heading metadata
Layer 2 — Machine Authority: YAML/JSON SSOT records with stable IDs and lineage
Layer 3 — Generated Outputs: Word, Excel, HTML, notebooks and reports
```

Authority remains in YAML/JSON. Word, Excel, HTML and notebooks are generated outputs.

## 3. Numbering policy

Visible Word outline numbers must not be stored as the source of truth.

Correct pattern:

```text
Stable ID = permanent identity
Heading level = structural position
Word numbering = generated presentation from the `.dotm` multi-level list
```

Example Markdown authoring form:

```markdown
# Requirements {#SEC-REQ-001 style="Heading 1"}
## Installation Requirements {#SEC-REQ-001-001 style="Heading 2"}
### Python Package Requirements {#SEC-REQ-001-001-001 style="Heading 3"}
```

The rendered Word output may show `1`, `1.1`, `1.1.1`, but those numbers are generated by the Word template.

## 7. Control principle

Every block must have:

- `block_id`
- `block_type`
- `title`
- `status`
- `version`
- `section.heading_id`
- `section.heading_level`
- `section.word_style`
- `lineage`
- `render`
- `content`

Any manual edit to generated Word or Excel should become a change request before being absorbed back into SSOT.
