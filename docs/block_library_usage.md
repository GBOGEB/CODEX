# CODEX Block Library Usage

This note explains the current MVP commands for the reusable block library.

## Validate block records

Run:

```bash
python scripts/validate_blocks.py ssot/blocks/sample_blocks.yaml
```

Expected result:

```text
PASS: ssot/blocks/sample_blocks.yaml contains 3 valid reusable block(s)
```

## Export sortable CSV register

Run:

```bash
python scripts/export_blocks_to_csv.py ssot/blocks/sample_blocks.yaml outputs/block_register.csv
```

Purpose:

- produce a sortable review table
- support opening the register in Excel
- avoid generated binary files in the first MVP

## Render Markdown preview

Run:

```bash
python scripts/render_markdown_preview.py ssot/blocks/sample_blocks.yaml outputs/block_library_preview.md
```

Purpose:

- produce a color-readable copy/paste view
- support GitHub and VS Code preview
- keep YAML as the source of truth

## Control rules

- Edit YAML first.
- Validate before rendering.
- Treat CSV and Markdown as generated outputs.
- Do not manually store Word outline numbers.
- Word numbering must be generated by the Word template.
